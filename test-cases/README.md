# Test Cases – Components Page

This folder contains the manual test case report for the **Components** category page of the LambdaTest E-Commerce Playground.

## File
- `TCs_Report_Component_Page.xlsx`

## Contents
The report includes **55+ test cases** covering:
- Header & Breadcrumb navigation
- Sub-categories display and navigation
- Toolbar (Grid/List view, Sort, Show, Compare)
- Product cards (hover actions, Add to Cart, Wishlist, Quick View)
- Pagination
- Filters (Price, Manufacturer, Sub-category, Search, Color, Availability, Size, Discount, Rating)
- Left Sidebar categories

## Test Case Structure
Each test case includes:
| Field | Description |
|---|---|
| Test Case ID | Unique identifier (e.g., TC-CH-001) |
| Title | What the test case verifies |
| Preconditions | State required before execution |
| Test Steps | Step-by-step actions |
| Test Data | Input data / URL used |
| Expected Result | What should happen |
| Actual Result | What actually happened (if different) |
| Status | Pass / Fail |
| Priority | High / Medium / Low |
| Requirement ID | Linked requirement |
