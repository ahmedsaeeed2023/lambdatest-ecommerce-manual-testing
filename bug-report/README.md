# Bug Reports – Components Page

This folder contains all bugs identified while manually testing the **Components** category page of the LambdaTest E-Commerce Playground.

## File
- `Bug_Report_Component_Page.xlsx`

## Summary
**20 bugs** were identified and documented, including:
- 4 Critical severity
- 11 High severity
- 5 Medium severity

## Bug Report Structure
Each bug report includes:
| Field | Description |
|---|---|
| Bug ID | Unique identifier (e.g., BUG-PC-001) |
| Title/Summary | Short description of the issue |
| Description | Detailed explanation |
| Steps to Reproduce | How to trigger the bug |
| Expected Result | What should happen |
| Actual Result | What actually happens |
| Severity | Critical / High / Medium |
| Priority | High / Medium |
| Status | Open |
| Reporter & Date | Who found it and when |
| Assigned To | Team responsible |
| Environment | Browser / OS / URL |
| Attachments/References | Linked test case ID(s) |
